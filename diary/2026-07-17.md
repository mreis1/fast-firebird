# 2026-07-17 — Session 40: npm publish machinery

npm org `fast-firebird` registered by Marcio → built everything between here
and the first `v0.1.0` release:

- **Per-package READMEs** (`packages/core/README.md`, `packages/drizzle/…`) —
  npm renders the package page from the *tarball's* README, so without these
  the pages would be blank. Core: positioning + highlights + quick start;
  drizzle: what-works list (incl. the honest `with:` boundary). Drizzle
  example verified against actual `index.ts` exports (`firebirdTable`,
  `integer`, `varchar`, `drizzle` — all real).
- **`publishConfig: { access: "public" }`** in both packages (scoped packages
  default to private publish, which fails without a paid plan). Provenance
  deliberately NOT in publishConfig — it comes from the CI env var, so a
  local emergency publish stays possible.
- **`.github/workflows/publish.yml`** — triggers on GitHub Release published;
  same pnpm setup + references-stub as ci.yml; build + core unit tests as a
  final gate (the full Firebird matrix is ci.yml's job on every push);
  `pnpm -r publish --access public --no-git-checks` with
  `NPM_CONFIG_PROVENANCE=true` + `id-token: write`. `-r` skips the private
  packages and **rewrites `workspace:*` to the real version** — which is why
  publishing must go through pnpm, never raw `npm publish` (drizzle's
  manifest would ship a broken `workspace:*` dependency).
- **Tarballs verified** (`npm pack --dry-run`): core = LICENSE, NOTICE,
  README, dist, package.json (567 kB, 10 files); drizzle = LICENSE, README,
  dist (47 kB, 9 files). No stray files.

Remaining, all user-side: decide on private-repo mentions in plans/diary →
flip repo public (npm provenance REQUIRES a public repo — Sigstore
